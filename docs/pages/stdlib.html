Standard library

The standard library is a set of ready-made functions grouped into namespaces: string, table, math, bit32, os, and coroutine. Each is a global table; you call a function with a dot, like string.upper("hi"). Every function is listed below with what it does. Positions in strings and arrays are 1-based: the first item is at index 1.

There is no method-call sugar on values: ("hi"):upper() does not work — always call through the namespace, string.upper("hi").

string

Functions for working with text. All positions count characters (Unicode code points), not bytes. There are no Lua patterns — searching and splitting use plain text. Where a string is expected, a number is accepted and converted automatically.

FunctionDescription
string.len(s)The number of characters in s. Same as #s.
string.sub(s, i, j?)The part of s from position i to j (inclusive). Negative positions count from the end (-1 is the last character); j defaults to -1. An empty range gives "".
string.upper(s)A copy of s with letters uppercased.
string.lower(s)A copy of s with letters lowercased.
string.rep(s, n, sep?)The string s repeated n times, with sep between copies when given. n of zero or less gives "".
string.reverse(s)The characters of s in reverse order.
string.byte(s, i?, j?)The numeric codes of the characters from position i to j, as multiple return values. i defaults to 1, j defaults to i (one code); negative i counts from the end.
string.char(...)A string built from the given character codes. An invalid code is an error.
string.find(s, sub, init?)The start and end positions of the first occurrence of sub in s at or after position init (default 1), or nil if not found. Plain text only — no patterns.
string.contains(s, sub)True if s contains sub.
string.startswith(s, prefix)True if s begins with prefix.
string.endswith(s, suffix)True if s ends with suffix.
string.trim(s)A copy of s with leading and trailing whitespace removed.
string.split(s, sep?)An array of the pieces of s split on each occurrence of sep. sep defaults to a single space; an empty sep splits into individual characters.
string.format(fmt, ...)A formatted string. Supported placeholders: %s (any value), %d / %i (integer), %x / %X (hex), %f / %g (float, with optional precision like %.2f), and %% for a literal percent. Unrecognized placeholders are copied through unchanged.
print(string.upper("luar"))                 -- LUAR
print(string.sub("hello", 1, 3))           -- hel
print(string.sub("hello", -3))             -- llo
print(string.split("a,b,c", ","))            -- { "a", "b", "c" }
print(string.rep("ab", 3, "-"))            -- ab-ab-ab
print(string.format("%s is %.2f", "x", 1.5)) -- x is 1.50

table

Functions for working with tables used as arrays. They operate on the contiguous array part (indices 1, 2, 3, ...); see Tables for how the array and hash parts work.

FunctionDescription
table.insert(t, value)Append value to the end of the array. Returns nothing.
table.insert(t, pos, value)Insert value at position pos (clamped into range), shifting later items up. On a fixed-size buff table that is already full, inserting is a "buff overflow" error.
table.remove(t, pos?)Remove and return the element at pos (default: the last element), shifting later items down. Returns nil when the array is empty or pos is out of range.
table.concat(t, sep?, i?, j?)Join the elements from index i to j (defaults: the whole array) into one string, separated by sep (default ""). Each element is converted to its string form.
table.sort(t, compare?)Sort the array part in place, ascending by <. With a comparator, compare(a, b) must return true when a should come before b. The sort is stable: equal elements keep their order. Returns nothing.
table.keys(t)A new array of the table's keys: the array indices 1..#t first, then the hash-part keys (in no particular order).
table.clear(t)Remove every entry (array and hash part) from the table in place, keeping its metatable. Returns nothing. Because the same table object is emptied, every other reference to it sees the now-empty table.
table.pack(...)Collect the given values into a new array table, with an extra field n holding the count.
table.unpack(t, i?, j?)Expand the elements from index i to j (defaults: the whole array) into multiple return values. There is no global unpack — only this one.
local t = { 3, 1, 2 }
table.insert(t, 4)
table.sort(t)
print(table.concat(t, ", "))   -- 1, 2, 3, 4
table.sort(t, function(a, b) return a > b end)   -- descending

local packed = table.pack("a", "b")
print(packed.n)                  -- 2
print(table.unpack(packed, 1, packed.n))   -- a   b

math

Numeric functions and constants. They accept any number; a string that looks like a number (such as "10") is converted automatically. Functions marked with results that "collapse to integers" return an integer whenever the mathematical result is whole (so math.floor(2.7) is the integer 2, not 2.0).

FunctionDescription
math.abs(x)The absolute value of x. Whole results collapse to integers.
math.ceil(x)The smallest integer not less than x (round up). Returns an integer.
math.floor(x)The largest integer not greater than x (round down). Returns an integer. Useful after /, which always produces a float.
math.round(x)x rounded to the nearest integer. Returns an integer.
math.sqrt(x)The square root of x. Whole results collapse to integers.
math.sin(x), math.cos(x), math.tan(x)Trigonometric functions; x is in radians. Return floats.
math.asin(x), math.acos(x)Inverse trigonometric functions. Return floats (radians).
math.atan(y, x?)The arc tangent of y, in radians. With two arguments it is the two-argument form (atan2): the angle of the point (x, y), using the signs of both to pick the quadrant.
math.exp(x)e raised to the power x. Returns a float.
math.log(x, base?)The natural logarithm of x, or the logarithm in the given base. Returns a float.
math.pow(x, y)x raised to the power y (same as x ^ y). Whole results collapse to integers.
math.fmod(x, y)The remainder of x / y, truncated toward zero (the result keeps the sign of x). Always a float. Note this differs from the % operator, which floor-divides.
math.modf(x)Two return values: the integer part of x (as an integer) and the fractional part (as a float).
math.max(...)The largest of the given numbers. At least one argument is required.
math.min(...)The smallest of the given numbers. At least one argument is required.
math.clamp(x, lo, hi)x limited to the range lo to hi.
math.sign(x)The integer -1, 0, or 1 depending on the sign of x.
math.deg(x)Radians converted to degrees (float).
math.rad(x)Degrees converted to radians (float).
math.random()A random float in the range 0 (inclusive) to 1 (exclusive).
math.random(m)A random integer from 1 to m inclusive. m less than 1 is an error ("interval is empty").
math.random(lo, hi)A random integer from lo to hi inclusive. hi < lo is an error.
math.randomseed(n)Seed the random generator so the sequence is repeatable. Returns nothing. The generator is per thread, so each coroutine has its own sequence.
ConstantValue
math.piThe ratio pi.
math.hugePositive infinity.
math.maxinteger / math.minintegerThe largest and smallest integers.

bit32

Bitwise operations on integers, treated as 32-bit unsigned values. Arguments are truncated to 32 bits; results are integers in the range 0 to 4294967295 (so bit32.bnot(0) is 4294967295, not -1).

FunctionDescription
bit32.band(...)Bitwise AND of all the arguments.
bit32.bor(...)Bitwise OR of all the arguments.
bit32.bxor(...)Bitwise exclusive OR of all the arguments.
bit32.bnot(a)Bitwise NOT (flip every bit).
bit32.lshift(a, n)Shift a left by n bits. n outside 0–31 gives 0.
bit32.rshift(a, n)Shift a right by n bits, filling with zeros. n outside 0–31 gives 0.
bit32.arshift(a, n)Arithmetic right shift: the sign bit is copied into the vacated bits. A negative n shifts left instead; n of 32 or more gives 0 (or 4294967295 when the sign bit was set).

os

FunctionDescription
os.time()The current time as an integer: seconds since the Unix epoch.
os.clock()A float: seconds elapsed since the program first called it. Useful for measuring how long something takes; subtract two readings.

There is no os.date.

coroutine

A coroutine is a function that can pause itself and be resumed later, keeping its place and its local variables. Coroutines share the globals of the script that created them.

FunctionDescription
coroutine.create(fn)Make a new coroutine that will run fn when first resumed. fn must be a function. Returns a thread value.
coroutine.resume(co, ...)Start or continue a coroutine. The arguments are passed to the function on the first resume, or become the result of the matching yield after that. Returns true followed by the yielded or returned values, or false followed by an error message (for example when resuming a dead or waiting coroutine).
coroutine.yield(...)Pause the current coroutine, handing the given values back to whoever resumed it. Returns whatever the next resume passes in. Yielding from the main thread is an error.
coroutine.status(co)The coroutine's state as a string: "suspended" (ready to resume), "running" (executing, including in the background), "waiting" (blocked inside a host call), or "dead" (finished or failed).
coroutine.running()The coroutine running right now, plus a boolean that is true when it is the main thread. The main thread is itself a coroutine, so you always get a thread back; inside a coroutine you get that coroutine and false.
coroutine.close(co)Finish a suspended coroutine so it can be cleaned up. Returns a boolean: whether it was closed.
local co = coroutine.create(function(a)
  print("start", a)
  local b = coroutine.yield(a + 1)   -- pause, hand back a+1
  print("resumed with", b)
end)
coroutine.resume(co, 10)   -- start: 10   then yields 11
coroutine.resume(co, 20)   -- resumed with: 20

Coroutines really run together

Unlike classic Lua coroutines, LUAR coroutines are not purely cooperative: each one runs on its own thread, coordinated by a lock so that only one piece of LUAR code executes at a time. Your code still behaves single-threaded, but a slow or blocked coroutine does not stop the others. Two things follow from this:

  • A long resume moves to the background. If a resume does not yield or return within a short grace period, coroutine.resume returns just true and the coroutine keeps running on its own; its status stays "running". Resuming it again waits for its next yield or return.
  • Blocking host calls detach. When a coroutine calls a host function that blocks (such as a wait or sleep provided by the embedding program), the coroutine detaches: its status becomes "waiting" and the rest of the program continues immediately. A waiting coroutine cannot be resumed by hand — coroutine.resume returns false, "cannot resume waiting coroutine" — it picks itself back up when the blocking work finishes.

The standalone runner (and the standard host setup) waits for all background and waiting coroutines to finish before the program exits, so work started this way is not cut short.

Global builtins

Besides the namespaces above, there are functions available directly without a namespace, such as print, type, pcall, pairs, and the reflection functions. See Builtins for the full list, and for how to override them.
