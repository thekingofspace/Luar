Global builtins

These functions are available everywhere without a require or a namespace. They are grouped below by what they do; every entry says what the function returns. All of them are ordinary global bindings, so they can be shadowed or replaced (see Overriding globals at the bottom).

Output

FunctionPurpose
print(...)Write each argument's display string to standard output, separated by tabs, ending with a newline. Honors a class operator tostring and a table __tostring metamethod. Returns nothing.
warn(...)Exactly like print, but writes to standard error instead.

Types and conversion

FunctionPurpose
type(v)The basic type name as a string: "nil", "bool", "number", "string", "table", "enum", "function", "thread", "class", or "interface". Both integers and floats are "number"; class instances are "table".
TypeOf(v)The "deep" type name as a string. Unlike type, it asks the value itself first — see TypeOf and the type protocol below. Falls back to the same name type would give.
tostring(v)The display string of any value. Honors operator tostring on class instances and the __tostring metamethod on tables; tables without one render inline like {1, 2}.
tonumber(v)Numbers pass through unchanged. Strings are trimmed and parsed as an integer first, then as a float; anything else (including unparseable strings) gives nil.
tonumber(s, base)Parse the string s as an integer in the given base (2–36); returns nil if it does not parse. A base outside 2–36, or a non-string value, is an error.

TypeOf and the type protocol

type(v) always answers with the built-in kind of a value, so every class instance is just "table". TypeOf(v) lets values describe themselves. It checks, in order:

  1. If v is a class instance (or a class value) whose class declares operator type(), that operator is called and must return a string.
  2. If v is a plain table with a __type entry in its metatable, that entry is used: a string is returned as-is, a function is called with the table and must return a string.
  3. Otherwise TypeOf returns the same name as type(v).
class Vec {
  public x: number = 0
  operator type() return "Vec" end
}

local v = Vec()
print(type(v))     -- table
print(TypeOf(v))   -- Vec

local p = setmetatable({}, { __type = "Point" })
print(TypeOf(p))   -- Point

If the operator or __type handler returns anything other than a string, TypeOf raises an error.

Errors

FunctionPurpose
assert(v, message?, ...)If v is truthy, returns all of its arguments (so it can wrap a call transparently). Otherwise raises message — a non-string message is converted to its display string — or "assertion failed!" when no message is given.
error(message)Raise message as a runtime error; a non-string value is converted to its display string. There is no level argument — any extra arguments are ignored.
pcall(fn, ...)Call fn with the given arguments, catching any error. Returns true followed by the function's results on success, or false followed by the error message string on failure.
local ok, err = pcall(function()
  error("boom")
end)
print(ok, err)   -- false  boom

There is no xpcall; use pcall and handle the message yourself.

Iteration and varargs

FunctionPurpose
ipairs(t)An iterator over the array part of t: pairs 1, t[1], 2, t[2], ... stopping at the first nil. Use in for i, v in ipairs(t) do.
pairs(t)Returns next, t, nil, iterating every entry: the array part first (in order), then the hash part (in no particular order).
next(t, key?)The key and value following key in t, or nil when there are no more. Passing nil (or nothing) starts from the beginning.
select("#", ...)The number of extra arguments, as an integer.
select(n, ...)The arguments from position n onward, as multiple values. A negative n counts from the end; 0 or an index past the end (when negative) is an error.

Metatables and raw access

See Metamethods for what a metatable can do.

FunctionPurpose
setmetatable(t, m)Set the metatable of table t to table m, or remove it when m is nil. Returns t. Errors if t is not a table or m is neither a table nor nil.
getmetatable(t)The metatable of t, or nil if it has none (or is not a table). There is no __metatable protection — you always get the real metatable.
rawget(t, k)Read t[k] without invoking __index. Returns the value, or nil when absent.
rawset(t, k, v)Write t[k] = v without invoking __newindex. Returns t.
rawequal(a, b)Equality without metamethods: numbers by value (ints and floats cross-compare), strings by content, tables/functions/classes by identity. Returns a boolean.
rawlen(v)The array length of a table or the character count of a string, ignoring any __len metamethod. Returns an integer.

Modules and memory

FunctionPurpose
require(path)Load a module and return its value (the module's first returned value). Each module runs once and is cached; later requires return the same value. Supports relative paths, @alias names, folder modules, and precompiled .luarb files — see Modules. A circular require is an error.
collectgarbage()Ask the garbage collector to run at the next safe point (between top-level statements) — it does not collect immediately. Returns nothing.

Class reflection

These inspect classes and instances; see Classes.

FunctionPurpose
instanceof(v, C)True if v is an instance of class C or one of its subclasses, or of a class that implements interface C. False for non-instances; errors if C is not a class or interface.
classname(v)The name (a string) of a class value or of an instance's class, otherwise nil.
classof(v)The class value of an instance, otherwise nil.
superclass(v)The parent class of a class or instance, or nil if it has none.
methodsof(v)An array of the method names of a class or instance, walking up the inheritance chain, without duplicates.
isabstract(v)True if the class (or the instance's class) is declared abstract.

What is deliberately not here

Coming from Lua, note these differences: there is no xpcall, no global unpack (use table.unpack), no coroutine.launch, and no os.date. String functions do not support Lua patterns — string.find is a plain substring search.

Overriding globals

The builtins are ordinary global bindings, so you can replace them with your own. This is useful for logging, sandboxing, or adapting LUAR to a host program. There are two ways, depending on how far the change should reach.

Replace it everywhere with pub

Declaring a pub binding with the same name replaces the builtin in the shared global scope, so every file sees your version. Capture the original first if you still need it.

pub const realPrint = print
pub print = function(...)
  realPrint("[log]", ...)
end

print("hi")   -- [log]  hi

The same works for any global, including require:

pub const realRequire = require
pub require = function(path)
  realPrint(`requiring {path}`)
  return realRequire(path)
end

Override it just for one file or block with local

A local binding with the builtin's name shadows it only inside the current scope. The rest of the program is unaffected. Capture the original into another local if you want to call through to it.

local realPrint = print
local print = function(x) realPrint("LOG: " .. tostring(x)) end

print("hello")   -- LOG: hello   (only in this scope)

From the host (Rust)

An embedding program can install or replace any global from Rust with set_global_fn or set_global. See the Host API.
