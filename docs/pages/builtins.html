Overriding globals

The builtins are ordinary global bindings, so you can replace them with your own. This is useful for logging, sandboxing, or adapting LUAR to a host program. There are two ways, depending on how far the change should reach.

Replace it everywhere with pub

Declaring a pub binding with the same name replaces the builtin in the shared global scope, so every file sees your version. Capture the original first if you still need it.

pub const realPrint = print
pub print = function(...)
  realPrint("[log]", ...)
end

print("hi")   -- [log]  hi

The same works for any global, including require:

pub const realRequire = require
pub require = function(path)
  realPrint(`requiring {path}`)
  return realRequire(path)
end

Override it just for one file or block with local

A local binding with the builtin's name shadows it only inside the current scope. The rest of the program is unaffected. Capture the original into another local if you want to call through to it.

local realPrint = print
local print = function(x) realPrint("LOG: " .. tostring(x)) end

print("hello")   -- LOG: hello   (only in this scope)

From the host (Rust)

An embedding program can install or replace any global from Rust with set_global_fn or set_global. See the Host API.
