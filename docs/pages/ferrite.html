Ferrite checker

Ferrite is the static checker. It analyzes a program without running it and reports warnings and errors. The editor shows them inline, and the runner prints them before executing.

Checks

CodeMeaning
UnusedVariableA declared variable is never used (prefix with _ to silence).
UnusedParameterA function parameter is never used.
UnusedLoopVariableA loop variable is never used.
RedeclarationA variable is redeclared in the same scope.
ShadowedVariableA variable shadows one from an outer scope.
UnreachableCodeCode follows a return or break in the same block.
EmptyBlockAn empty do, if, while, case, or default body.
EmptyClassA class with no members.
SelfAssignmentA variable is assigned to itself.
MutateImmutableAn immutable (const) variable is changed (assigning nil is allowed).
BreakOutsideLoopA break is not inside a loop (error).
ConstantConditionA condition is always true or always false.
RedundantBooleanComparing with a boolean literal, such as x == true.
DoubleNegationnot not x is redundant.
DuplicateClassMemberA class member is declared more than once.
DuplicateTableKeyA table key is set more than once.
DuplicateCaseValueA switch case value appears more than once.
SelfComparisonA variable is compared with itself.
DivisionByZeroDivision or modulo by a literal zero.
AbstractMethodWithBodyAn abstract method has a body.
FinalAndAbstractA method is both abstract and final.
SyntaxErrorThe file does not parse (error).

Directives

A directive is a comment beginning with --#. Use it to silence checks. Names can be comma or space separated, and all silences everything.

--#disable UnusedVariable             # whole file
--#disable MutateImmutable, UnusedVariable   # several at once
--#disable all                         # everything

x = 2  --#disable-line MutateImmutable   # just this line

--#disable-next-line MutateImmutable     # the following line
x = 3
