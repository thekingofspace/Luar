Ferrite checker

Ferrite is the static checker. It analyzes a program without running it and reports warnings and errors. The editor runs it live on every .luar file (see Editor & LSP), and any Rust host can run the same checks with one function call.

Running Ferrite from Rust

Ferrite is one function. luar::ferrite::check takes source text and returns every diagnostic it found, already sorted by line. It tokenizes and parses the source itself, so you hand it a raw string — there is no separate parse step to wire up.

use luar::ferrite::{check, Severity};

let src = std::fs::read_to_string("main.luar")?;
let diagnostics = check(&src);   // Vec<Diagnostic>

for d in &diagnostics {
    // Display gives: "2:warning [MutateImmutable] cannot change immutable variable 'x'"
    println!("{d}");
}

Each Diagnostic exposes its parts so you can render or filter them yourself, and its Display produces the one-line form shown above.

FieldTypeWhat it is
severitySeveritySeverity::Warning or Severity::Error. Only BreakOutsideLoop and SyntaxError are errors; every other check is a warning.
code&'static strThe check name, such as "MutateImmutable" (see the table below).
messageStringA human-readable explanation.
lineu32The 1-based source line. A parse error carries the failing line; 0 means the lexer itself failed.

Ferrite never decides for you whether to stop — it only reports. A typical runner checks first, prints everything, and refuses to run only if something is an error, letting warnings through:

let diagnostics = check(&src);
let has_error = diagnostics.iter().any(|d| d.severity == Severity::Error);
for d in &diagnostics {
    eprintln!("{d}");
}
if has_error {
    return Err("refusing to run: Ferrite reported errors".into());
}
let mut i = luar::Interpreter::new();
i.run_source(&src)?;

A file that does not tokenize or parse comes back as a single diagnostic with code "SyntaxError" and Severity::Error, so the same call that catches lint issues also catches a file that will not compile. The in-source --#disable directives below are honored by check as well, and the full list of lint codes is available at run time as luar::ferrite::CHECKS.

Checks

Every check is a warning except BreakOutsideLoop and SyntaxError, which are errors. Names starting with _ are exempt from all three unused checks, and pub bindings are never flagged unused (they exist for other files). Loop lints point at the for line itself.

CodeSeverityFires on
UnusedVariablewarningA declared variable that is never read: local count = 5 and nothing uses count.
UnusedParameterwarningA function parameter that is never read: function f(extra) return 1 end.
UnusedLoopVariablewarningA loop variable that is never read: for i = 1, 10 do print("x") end.
RedeclarationwarningThe same name declared twice in one scope: local x = 1 then local x = 2.
ShadowedVariablewarningA declaration hiding an outer variable: local x = 1 then do local x = 2 ... end.
UnreachableCodewarningA statement after return or break in the same block: return 1 followed by print(2).
EmptyBlockwarningAn empty do, if, else, while, case, or default body: do end.
EmptyClasswarningA class with no members at all: class C {}.
SelfAssignmentwarningAssigning a variable to itself: x = x.
MutateImmutablewarningReassigning an immutable (const) variable: const x = 1 then x = 2. Assigning nil is allowed — it frees the binding.
BreakOutsideLooperrorA break with no enclosing loop — function bodies start a fresh loop context, so a break inside a callback in a loop still fires.
ConstantConditionwarningA literal true/false condition: if true then ... end, while false do ... end.
RedundantBooleanwarningComparing with a boolean literal using == or ~=: x == true, x ~= false.
DoubleNegationwarningnot not x — redundant.
DuplicateClassMemberwarningA field, method, getter, or setter declared twice in one class: class C { x = 1 x = 2 } (a getter and setter of the same name are fine — they count separately).
DuplicateTableKeywarningA literal key set twice in one table constructor: { a = 1, a = 2 }. Literal string, number, and boolean keys are tracked.
DuplicateCaseValuewarningTwo case patterns with the same literal value in one switch: case 2 ... end twice.
SelfComparisonwarningA variable compared with itself with any comparison operator: x == x, x < x.
DivisionByZerowarning/ or % with a literal zero on the right: x / 0, x % 0.0.
AbstractMethodWithBodywarningAn abstract method that has statements in its body: abstract function f() return 1 end.
FinalAndAbstractwarningA method marked both abstract and final — it can neither be used nor overridden.
SyntaxErrorerrorThe file does not tokenize or parse. Returned alone — no other checks run on a broken file.

Editor-only checks

The language server adds its own diagnostics on top of Ferrite (described in full on the Editor & LSP page). They honor exactly the same directives, so these codes work in --#disable lines too:

CodeSeverityFires on
MutateImmutableerrorThe editor's own version of the check above, aware of .luard ambient globals' mutability. It replaces Ferrite's in the editor.
DuplicateEnumVarianterrorThe same variant name twice in one enum declaration.
GenericArityerrorA generic type used with the wrong number of arguments: Box where Box<T> is required.
UnknownClasswarningAn extends/mixin target no file declares (names are case-sensitive).
RequireCyclewarningA require that closes a cycle — the runtime errors on these.
NotNilerrorDeclaring a NotNil<T> binding without a non-nil initializer, assigning literal nil to one, or passing literal nil to a NotNil parameter.
FinalOverrideerrorExtending a final class or overriding a final method — both error at runtime.

Directives

A directive is a comment beginning with --#. Use it to silence checks. After the keyword, list check names separated by commas and/or spaces — no braces, no quotes. The special name all silences everything. Three forms exist:

FormScope
--#disable Code1, Code2The whole file, on its own line or anywhere in one.
--#disable-line Code1, Code2Only the line the directive is on — written at the end of the offending line.
--#disable-next-line Code1, Code2Only the line directly below the directive.
--#disable UnusedVariable                    -- whole file
--#disable MutateImmutable, UnusedVariable   -- several at once
--#disable all                               -- everything

x = 2  --#disable-line MutateImmutable       -- just this line

--#disable-next-line MutateImmutable         -- the following line
x = 3

The directive is found by scanning each line for the --#disable text, so it works inside a longer comment too. In the editor, typing --# autocompletes the three keywords, and after one of them the full list of check names — Ferrite's and the editor-only codes — is offered.
