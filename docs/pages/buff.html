Buffs (manual memory)

A buff is a third kind of binding alongside local and const. It is a mutable, fixed-size variable whose lifetime you manage by hand: unlike an ordinary local, a buff is never collected by the garbage collector and never goes out of scope. It stays alive until you release it explicitly with freebuff. Buffs exist for data you want to keep under manual control with a predictable lifetime.

Declaring a buff

Write buff, a fixed size, a name, and an initial value. The size is the capacity of the buffer (in bytes for string contents).

buff 32 name = "Player_1"
print(name)            -- Player_1

Buffs are mutable but fixed-size

You can reassign a buff as many times as you like, but a value that does not fit its declared size is rejected with a run-time error. This is what "fixed size" means. For a string, the size is the number of bytes.

buff 8 tag = "ok"
tag = "changed"     -- fine: 7 bytes fits in 8
tag = "way too long" -- run-time error: exceeds the fixed size 8

Tables, class instances, and native objects

When a buff holds a table, a class instance, or a native object, the size is the number of entries it may contain, and the limit is enforced on every growth, not just on assignment. Adding an element with table.insert or an index, or adding a field to an instance, fails once the buffer is full. Overwriting an element or field that already exists is always allowed, because it does not grow the buffer.

buff 3 items = {1, 2}
table.insert(items, 3)   -- fine: now holds 3
table.insert(items, 4)   -- run-time error: would exceed the fixed size 3
items[1] = 99            -- fine: overwriting, not growing

class Vec { public x: number = 0 }
buff 2 v = Vec()
v.y = 1                  -- fine: instance now has 2 fields
v.z = 2                  -- run-time error: would exceed the fixed size 2

A buff can never be pub

Buffs are always private to the file that declares them. Marking one pub is a compile error, because a manually managed buffer must not escape to importers.

pub buff 8 x = "a"   -- error: a buff cannot be `pub`

nil does not free a buff

Assigning nil to an ordinary variable makes its old value eligible for collection. A buff is different: setting it to nil does not free it. The buffer stays allocated and you can keep writing to it. The only way to reclaim a buff is freebuff.

buff 16 slot = "data"
slot = nil            -- still allocated, just holds nil now
slot = "more data"     -- the same buff is reused

Buffs never go out of scope

A buff declared inside a block or a function outlives it. After the enclosing scope ends, the buff is still there. This is the defining property of a buff: its lifetime is not tied to the block it was created in.

local function setup()
  buff 8 cache = "warm"
end
setup()
print(cache)           -- warm  (the buff survived the function call)

Freeing with freebuff

freebuff name releases a buff immediately, bypassing the garbage collector entirely — the memory goes straight to freed. After a buff is freed, its name no longer exists.

buff 8 handle = "open"
freebuff handle      -- released now, not at end of scope, not by the GC

Use-after-free is an error

Because freeing removes the name, reading a buff after freebuff is a run-time error rather than returning stale data. This is deliberate: it makes manual memory safe by catching use-after-free instead of handing back garbage.

buff 8 p = "x"
freebuff p
print(p)              -- run-time error: undefined variable 'p'

freebuff also rejects names that are not buffs, so you cannot accidentally free a normal local or const.

Buffs work in precompiled bytecode

Buffs and freebuff are preserved by precompilation, so a .luarc file that uses them behaves exactly like the source.

When to use a buff

  • You need a value with a deterministic lifetime that you release at an exact point, rather than whenever the collector runs.
  • You want a fixed-capacity slot that rejects oversized data.
  • You want the value to persist past the block that created it without making it a global.

For everything else, prefer an ordinary local or const and let the garbage collector manage memory for you.
