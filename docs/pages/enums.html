Enums

An enum is a set of named integer constants, accessed through the enum name with a dot: EnumName.Variant.

enum State { Idle Running Done }
print(State.Idle, State.Running, State.Done)   -- 0  1  2

Variants auto-increment from 0. An explicit = value is allowed, and later auto variants continue counting from it. Variants are separated by newlines, commas, semicolons, or plain whitespace — whichever reads best.

enum Status { Active = 10  Inactive  Banned = 99 }   -- 10, 11, 99
enum Flags { A, B, C }                              -- commas work too: 0, 1, 2

What an enum is at run time

An enum is a table whose entries are the variants, specially flagged so that type() can tell it apart from an ordinary table. The enum name is a constant binding — you cannot reassign it. Reading a variant that was never declared gives nil, like any missing table field, so a typo in a variant name does not error by itself; comparisons against it are simply never true.

enum State { Idle Running Done }
print(type(State))        -- enum
print(type(State.Idle))   -- number
print(State.Idel)         -- nil (typo: no error, just nil)

Adding to an enum and visibility

Declaring an enum whose name already exists does not replace it — the new variants are merged into the existing enum, and auto-numbering continues after the highest integer value already present. This lets one file (or one part of a file) extend an enum declared in another.

enum Color { Red Green Blue }   -- 0, 1, 2
enum Color { Yellow }           -- merged in as 3
print(Color.Yellow)             -- 3

Visibility follows the usual rules, with one twist: an enum is always hoisted to the top level of its module, even when the enum statement sits inside a function or a block. A plain enum is visible everywhere in the declaring module but not outside it; pub enum (or the equivalent export enum) makes it visible to every other module as well.

pub enum GameState { Menu Playing Paused }   -- visible to other files

function setup()
  enum Mode { Fast Safe }   -- still module-wide, despite being declared here
end
setup()
print(Mode.Safe)   -- 1

Editor check: duplicate variants

Listing the same variant twice inside one declaration is almost always a mistake (the second silently overwrites the first), so the editor flags it with the DuplicateEnumVariant check: duplicate variant 'Red' in enum 'Color'. Like every check it can be silenced with a --#disable DuplicateEnumVariant directive (see the linting page).

Using an enum as a type

An enum name is also a type. Annotating a variable, field, or parameter with an enum name says it should hold one of that enum's variants. This documents your intent and lets the editor offer the variants when you assign or compare. The value is still just the underlying number at run time.

pub enum GameState { Menu Playing Paused }

-- a variable typed as the enum
local state: GameState = GameState.Menu

-- a class field typed as the enum
class Game {
  public state: GameState = GameState.Menu
  function start() self.state = GameState.Playing end
}

-- a parameter typed as the enum
function describe(s: GameState)
  return switch(s)
    case GameState.Menu     return "in the menu" end
    case GameState.Playing  return "playing" end
    default                 return "paused" end
  end
end

print(describe(state))   -- in the menu
