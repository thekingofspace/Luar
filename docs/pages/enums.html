Enums

An enum is a set of named integer constants, accessed through the enum name.

enum State { Idle Running Done }
print(State.Idle, State.Running, State.Done)   -- 0  1  2

Variants auto-increment from 0; explicit values are allowed and later auto variants continue from there. Variants are separated by newlines, commas, or whitespace.

enum Status { Active = 10  Inactive  Banned = 99 }   -- 10, 11, 99

Adding to an enum and visibility

Declaring an existing enum adds variants to it instead of replacing it. A plain enum is module-global but private; a pub enum is visible to other modules.

enum Color { Red Green Blue }   -- 0, 1, 2
enum Color { Yellow }           -- 3

pub enum GameState { Menu Playing Paused }   -- visible to other files

Using an enum as a type

An enum name is also a type. Annotating a variable, field, or parameter with an enum name says it should hold one of that enum's variants. This documents your intent and lets the editor offer the variants when you assign or compare. The value is still just the underlying number at run time.

pub enum GameState { Menu Playing Paused }

-- a variable typed as the enum
local state: GameState = GameState.Menu

-- a class field typed as the enum
class Game {
  public state: GameState = GameState.Menu
  function start() self.state = GameState.Playing end
}

-- a parameter typed as the enum
function describe(s: GameState)
  return switch(s)
    case GameState.Menu     return "in the menu" end
    case GameState.Playing  return "playing" end
    default                 return "paused" end
  end
end

print(describe(state))   -- in the menu
