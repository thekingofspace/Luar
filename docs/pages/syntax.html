Syntax

Comments

-- a line comment runs to the end of the line

--[[
  a block comment can
  span many lines
]]

A comment directly above a declaration becomes its documentation and is shown on hover. A comment containing --# is a checker directive, not documentation. Directives silence Ferrite and editor diagnostics by code name:

--#disable UnusedVariable, ShadowedVariable
-- the directive above silences those two codes in the whole file

local kept = 1 --#disable-line UnusedVariable

--#disable-next-line all
local scratch = 2

The three forms are --#disable (whole file), --#disable-line (its own line), and --#disable-next-line. Codes are separated by spaces or commas, and everything after the directive keyword is read as code names, so do not put other text on a directive line. The special code all silences every check. The full list of codes is in the Ferrite chapter.

1_000_000         -- underscores may group digits (but not exponent digits)
"text"  'text'    -- strings (double or single quotes)

Strings

LUAR has three kinds of string. If you have not used a language with these before, here is what each one is for.

1. Quoted strings

The everyday string. Use double quotes "..." or single quotes '...' (they behave the same). A backslash starts an escape sequence, which is a short code for a character you cannot easily type, like a newline.

local a = "hello"
local b = 'it is fine to use single quotes'
local c = "line one\nline two"   -- \n is a newline

The supported escape sequences:

EscapeMeaning
\nNewline.
\tTab.
\rCarriage return.
\0The null character.
\a \b \f \v \eBell, backspace, form feed, vertical tab, escape.
\\A literal backslash.
\" \' \`A literal quote or backtick.
\xHHA byte written as two hexadecimal digits, for example \x41 is A.
\u{...}A Unicode code point, for example \u{1F600}.
\ then a newlineA line continuation: the backslash and newline are dropped.

Any other escape, such as \z, just produces the character after the backslash.

2. Multiline strings

Wrap text in double square brackets [[ ... ]] to write a string that spans many lines with no escaping at all. Everything between the brackets is taken literally, so quotes and backslashes need no escaping. A single newline right after the opening [[ is dropped, which keeps the first line clean.

local text = [[
This can span
many lines. "Quotes" and \backslashes are literal here.
]]

If your text itself contains ]], use a longer bracket with equals signs. The string ends only on a matching close bracket with the same number of equals signs.

local tricky = [==[ contains ]] safely ]==]

3. Interpolated strings

Wrap text in backticks ` ... ` to build a string with values spliced into it. Anything inside curly braces { } is an expression that is evaluated and inserted as text. This is usually cleaner than joining pieces with ...

local name = "world"
local count = 3
print(`Hello {name}, you have {count} messages`)
print(`{count} doubled is {count * 2}`)
print(`upper: {string.upper(name)}`)

The expression in { } can be anything: a variable, arithmetic, a function call, a field access. Each value is converted to text with tostring, so any value type works. To write a literal brace, escape it with \{ or \}. Interpolated strings can span multiple lines and support all the escape sequences above.

local s = `a literal brace: \{not interpolated\}, but {name} is`

Precedence

From loosest to tightest binding. Operators on the same row bind equally and group left to right unless noted.

LevelOperators
1 (loosest)or
2and
3== ~= < <= > >=
4.. (groups right: a .. b .. c is a .. (b .. c))
5+ -
6* / %
7unary not # -
8 (tightest)^ (groups right, and binds tighter than unary minus: -x^2 is -(x^2))

Calling without parentheses

When the single argument is a string literal, an interpolated string, or a table constructor, the parentheses may be dropped:

print "hello"          -- same as print("hello")
print `hi {name}`      -- same as print(`hi {name}`)
configure { debug = true }   -- same as configure({ debug = true })
A bare expression is only a statement when it is a call (including the call sugar forms above) or a switch. A value on a line by itself, such as 1 + 1, is a syntax error. Semicolons between statements are allowed and ignored.

Keywords

Every keyword in LUAR, grouped by what it does. Keywords are recognized by their position in the grammar rather than reserved outright, but you should treat them all as reserved: reusing one as a variable name will usually be misparsed.

Declaring and freeing variables

KeywordMeaning
localDeclares a mutable variable, visible from its declaration to the end of the enclosing block.
constDeclares a constant binding. It cannot be reassigned; only assigning nil may clear it.
pubExports a declaration as a module global, so files that require this one can see it.
exportAccepted before a declaration as a synonym for exporting at module scope.
buffDeclares a fixed-size, manually managed buffer: a mutable variable that never goes out of scope and is released only by freebuff. It can never be pub. See Buffs.
freebuffImmediately releases a buff, bypassing the garbage collector. Reading the name afterwards is an error.

Class member access and modifiers

KeywordMeaning
publicA class member reachable from anywhere. This is the default if no access is written.
privateA member reachable only from the declaring class's own methods.
protectedA member reachable from the declaring class and its subclasses.
staticA member that belongs to the class itself rather than to instances; reached through the class name.
abstractOn a class, makes it impossible to instantiate. On a method, declares it with no body that a concrete subclass must provide.
finalA method that cannot be overridden, or a class that cannot be extended.
overrideStates that a method replaces an inherited one. It is an error if there is nothing to override, which catches typos.

Declaring types and members

KeywordMeaning
functionDefines a function: standalone, local, or a method inside a class.
classDefines a class, a blueprint that objects are created from.
interfaceDefines a contract: a set of method names a class can promise to provide.
enumDefines an enumeration, a group of named constant values.
typeDefines a type alias, a name for a type expression.
extendsNames the single parent class that a class inherits from.
mixinFolds the members of one or more other classes into a class.
implementsDeclares the interfaces a class satisfies.
constructorThe special method that runs once when a class is instantiated.
destructorThe special method that runs when the last remaining reference to an instance disappears.
operatorOverloads an operator for a class, for example operator +.
getDefines a getter: a method that runs when a property is read.
setDefines a setter: a method that runs when a property is assigned.

Control flow

KeywordMeaning
ifBegins a conditional. Its branch runs when the condition is truthy.
thenSeparates an if or elseif condition from its body.
elseifAn additional condition tested when the previous ones were false.
elseThe branch that runs when no preceding condition matched.
endCloses a block: an if, loop, function, switch case, or do block.
forBegins a loop: numeric (for i = a, b) or generic (for k, v in iter).
whileBegins a loop that repeats while its condition stays truthy.
doOpens the body of a for/while loop, or a standalone block that makes a new scope.
inSeparates the loop variables from the iterator in a generic for.
breakExits the nearest enclosing loop immediately.
returnReturns zero or more values from a function.
switchBegins a multi-way branch on a single value.
caseA branch of a switch matched against the value.
defaultThe switch branch that runs when no case matched.

Logical operators

KeywordMeaning
andShort-circuit logical AND. Yields the first falsey operand or the last operand.
orShort-circuit logical OR. Yields the first truthy operand or the last operand.
notLogical negation: turns a truthy value into false and a falsey value into true.

Special values and identifiers

KeywordMeaning
selfInside a method, the object the method was called on.
superInside a method, the parent class, used to call an overridden method.
true / falseThe two boolean values.
nilThe absence of a value. Everything except nil and false is truthy.
...Inside a function declared with a trailing ... parameter, the extra arguments that were passed. In the last position of an expression list it expands to all of them; anywhere else it yields the first.
