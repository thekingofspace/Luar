Modules

A module is simply a file. As your program grows you split it into several files, and one file uses another by requiring it. require loads and runs another file, then hands back whatever that file returned. Paths are relative to the file doing the requiring, and each module is run only once: the result is cached and reused on later requires.

local shapes = require("./shapes")   -- loads ./shapes.luar next to this file

The leading ./ means "next to this file". You can reach into subfolders ("./ui/button") or up a level ("../shared"). The .luar extension is added for you.

What is shared

pub declarations are published to the shared global scope, so any file that requires the module can use them. local and const declarations stay private to the module. A module can also return a value, which becomes the result of require.

-- greeting.luar
pub const SHARED_VERSION = "1.0"   -- visible everywhere once required
const SECRET = "hidden"            -- private to this module

local M = {}
function M.hello(name) return "hi, " .. name end
M.answer = 42
return M
-- main.luar
local g = require("./greeting")
print(g.hello("luar"))   -- the editor completes g.hello and g.answer
print(SHARED_VERSION)    -- available because it is pub

Folder modules: init.luar

A folder can be a module. If a folder contains a file named init.luar, you can require the folder by name and that init.luar is what runs. This lets you group a feature into its own folder and present a single entry point.

ui/
  init.luar      # this runs when you require("./ui")
  button.luar
  slider.luar
local ui = require("./ui")   -- runs ui/init.luar

When you require a path, LUAR looks for <path>.luar first, then falls back to <path>/init.luar.

require("@self"): a folder's children

Inside an init.luar (or any file), require("@self") returns a table of every sibling module in the same folder. Each name.luar file (except init.luar) and each subfolder that has its own init.luar becomes an entry, keyed by its name and already required. This makes it easy for an init.luar to gather up its folder's pieces.

-- ui/init.luar
local parts = require("@self")   -- { button = ..., slider = ... }
return {
  Button = parts.button,
  Slider = parts.slider,
}
