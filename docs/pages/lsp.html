Language server (LSP)

LUAR ships a language server, written in Rust, that powers the VS Code extension. It reuses the same lexer, parser, and Ferrite checker as the runtime, so the editor and the language always agree. This page explains everything the server provides.

Installing

Install the LUAR extension in VS Code. It bundles the language server binary and starts it automatically for .luar and .luard files. No configuration is required.

Diagnostics

Every keystroke runs Ferrite and underlines problems in place, with the check code attached (for example MutateImmutable or UnusedVariable). See Ferrite checker for the full list and for the --#disable directives. Declaration files (.luard) are never linted.

Completion

The server is context aware. What it offers depends on where the cursor is.

  • Plain position: keywords, snippets, builtins, the standard library namespaces, declared variables and globals, class names, enum names, and type names. Function-like items insert their parentheses for you.
  • After a dot or colon on an instance or self: the class members (methods after a colon, all members after a dot), walking the extends chain and any mixins.
  • After a library name such as math. or string.: that library's functions and constants.
  • After an enum name such as Color.: the enum's variants, respecting module privacy.
  • After a required module such as M.: the module's exported types (classes, enums, type aliases) and the fields of the table it returns, read fresh from disk.
  • In a type position (after : or ::, or a class field): primitives, classes, enums, type aliases, and the type functions such as classof<...> and keyof<...>.
  • Inside require("..."): the workspace's other .luar files as relative paths, scanned fresh and de-duplicated. Accepting one replaces the path you have typed.
  • Comparing or assigning a literal-union variable: the exact allowed string values (for example "on" and "off").
  • After --#: the directive forms (disable, disable-line, disable-next-line), and after the directive word, every Ferrite check name plus all.

Hover and documentation comments

Hovering a symbol shows its kind and type. If a documentation comment sits directly above the declaration, it is shown in the hover box. This works for functions, variables, globals, classes, enums, and module exports, including across modules. Markdown is rendered, and lines beginning with @ are highlighted as tags.

-- The maximum health a player can have.
-- @type number
pub const MaxHP = 100

--[[
  Greet a person.

  Supports **Markdown** in the hover box.
  @param name the person
  @return string
]]
function greet(name) return "hi, " .. name end

A require binding hovers as the module it imports, and as an error type if no module of that name exists.

Inlay hints

The server shows the inferred type and mutability inline at a declaration when you have not written an annotation yourself, so you can see what the type solver concluded without cluttering the source.

Semantic highlighting

On top of the TextMate grammar, the server emits semantic tokens so coloring follows meaning rather than just shape:

  • Declaration keywords (local, const, pub, function, class, enum, and the access words) share one color.
  • extends, mixin, and implements are colored like class.
  • Class names in declarations and after extends/mixin/ implements/enum are highlighted distinctly.
  • Enum variants inside an enum body are colored as constants, not as types, so a variant named unknown is not mistaken for the primitive type.
  • Union and intersection type names (A | B & C) all color as types.
  • Library namespaces (math, string, ...) get their own color.

Type solving

The server runs a lightweight type solver to drive completion and hover. It resolves type aliases, function return types, classof and the other type functions, enum types, and module member types. Where a variable's type points at a class, member completion on that variable lists the class's members. See the complete list of type functions.

Declaration files

A .luard file describes globals the host provides from Rust. The server treats everything in it as an ambient global for completion and hover and never lints or runs it. This is how you give the editor types for native functions, tables, and classes you inject. See Modules and the Host API.

Editing help

Pressing Enter after an opener completes the block for you: if and while get their then or do plus a matching end, and a callable header that ends in ) (function, constructor, operator, get, set) gets its end. The server does not add a duplicate end when the block is already closed.
