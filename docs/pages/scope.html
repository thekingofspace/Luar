Scope

Scope is the part of a program where a name is visible. When you declare a variable, it can be seen from its declaration to the end of the block it lives in, and no further. Understanding scope is how you reason about which variable a name refers to and when a value can be freed.

Blocks make scopes

A block is a region of code introduced by a keyword and closed by end (or } for a class body). The body of an if, while, for, do, a function, a class method, and each case of a switch is its own block, and therefore its own scope.

local a = 1
do
  local b = 2     -- b lives only inside this do block
  print(a, b)      -- 1  2  (a is visible from the outer scope)
end
print(a)          -- 1
-- print(b) here would be an error: b is out of scope

Inner scopes can see outer ones

Code in an inner block can read and write variables declared in any enclosing block. The visibility flows inward, not outward.

local total = 0
for i = 1, 3 do
  total = total + i   -- the loop body can see and update total
end
print(total)         -- 6

Shadowing

If you declare a new variable in an inner scope using a name that already exists in an outer scope, the inner one hides (shadows) the outer one for the rest of that block. The outer variable is untouched and comes back into view when the inner block ends. The static checker warns about this with ShadowedVariable, because it is sometimes a mistake.

local n = 1
do
  local n = 2   -- a new variable that shadows the outer n
  print(n)      -- 2
end
print(n)        -- 1  (the outer n was never changed)

Loop variables

The variables of a for loop are scoped to the loop body. They do not exist after the loop ends. If you do not use a loop variable, name it _ to say so on purpose and silence the UnusedLoopVariable check.

for _, value in ipairs(list) do
  print(value)
end

Function and module scope

A function's parameters and the variables declared in its body are local to that function. At the top level of a file, declarations live in the file's (module) scope, which is private to that file unless you mark them pub.

-- in one file
const SECRET = "hidden"     -- private to this file
pub const VERSION = "1.0"  -- shared with any file that requires this one

See Variables for the difference between local, const, and pub, and Modules for how pub crosses file boundaries.

Closures capture their scope

A function created inside another function remembers the variables that were in scope where it was created, even after the outer function has returned. This captured environment is what makes a closure. Each call that creates a closure gets its own copy of those captured variables.

local function counter(start)
  local n = start
  return function()
    n += 1       -- still has access to n after counter returned
    return n
  end
end

local a = counter(0)
local b = counter(10)
print(a(), a(), b())   -- 1  2  11  (a and b have separate copies of n)

Scope and the garbage collector

When a scope ends, its local variables are no longer reachable through that scope. If nothing else refers to the value (no closure, no table, no global holds it), the garbage collector is free to reclaim it. Assigning nil to a binding also drops its reference early. See Garbage collection.

self and super inside methods

Inside a class method, two names are always in scope: self, the object the method was called on, and super, used to reach the parent class. Neither exists outside a method. See Classes.
