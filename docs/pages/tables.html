Tables

Tables are the general-purpose structure: arrays, maps, and objects are all tables. Arrays are 1-based.

local array = { 10, 20, 30 }
local map = { name = "luar", version = 1 }
local mixed = { 1, 2, [10] = "ten", label = "x" }

print(array[1])     -- 10
print(map.name)     -- "luar"  (sugar for map["name"])
print(#array)       -- 3

Constructors

A table constructor mixes three kinds of entry, separated by , or ; (a trailing separator is fine):

  • Positional values fill the array part in order, starting at index 1.
  • name = value sets a string key. The name can be any identifier — Luar has no reserved words, so even { local = 1, function = 2 } works, and reads back as t.local and t.function.
  • [expr] = value computes the key, for anything that is not a plain name: numbers, strings with spaces, or other values entirely.
local t = {
  "first",                  -- t[1]
  answer = 42,              -- t.answer
  [10] = "ten",             -- t[10]
  ["two words"] = true,    -- t["two words"]
}

If the last positional entry is a function call or ..., all of its values are appended:

local pair = function() return 8, 9 end
local t = { 7, pair() }   -- { 7, 8, 9 }

Keys can be (almost) any value

Table keys are not limited to strings and integers. Booleans, tables, functions, classes, and coroutines all work as keys. The rules:

  • Strings compare by content: t["ab"] and t["a" .. "b"] are the same slot.
  • Numbers: a float with no fractional part is treated as the matching integer, so t[2.0] and t[2] are the same slot. A float with a fractional part is an invalid key.
  • Tables, functions, classes, and coroutines compare by identity: two distinct empty tables are two distinct keys, even though they look equal. The table holds a reference to such a key, so the key is kept alive as long as the entry exists.
  • nil is never a valid key. Using nil (or a fractional float) as a key raises invalid table key of type .... Reading with an invalid key just gives nil.
local k1 = {}
local k2 = {}
local reg = {}
reg[k1] = "first"
reg[k2] = "second"
print(reg[k1], reg[k2])   -- first  second  (identity, not contents)

local handlers = {}
handlers[print] = "the print function"   -- functions as keys

Array part, map part, and #

Internally a table has two parts: a contiguous array part for integer keys 1, 2, 3, … with no gaps, and a map part for everything else. You normally never notice, but two things depend on it:

  • #t is the length of the array part only. Named keys and scattered integer keys do not count.
  • Assigning to index #t + 1 appends to the array part; this is what table.insert(t, v) does. ipairs walks the array part in order, while pairs visits every key in both parts.
local t = { 10, 20 }
t[#t + 1] = 30          -- append; t is now { 10, 20, 30 }
t.label = "x"
print(#t)               -- 3   (label does not count)

table.clear(t) empties both parts in place while keeping the table's identity and its metatable — useful when other code holds references to the same table. See the standard library for the full table module (insert, remove, concat, sort, unpack, …).

Metatables

A metatable is a second table that controls how the first table behaves. By itself a plain table just stores keys and values. Attach a metatable to it, and you can change what happens when someone reads a missing key, calls the table, turns it into a string, or uses it with an operator like +.

Set a metatable with setmetatable(t, meta) (it returns t; pass nil to remove the metatable) and read it back with getmetatable(t). The special fields you put in the metatable are called metamethods, and their names all begin with two underscores.

local base = { greet = function() return "hi" end }
local obj = setmetatable({}, { __index = base })
print(obj.greet())   -- found through __index

Metamethods in full

Each metamethod below lists when it runs, the arguments your function receives, and what it should return. Classes are built on this machinery: when you write operator + or operator tostring in a class, those become the matching metamethods for you, so you rarely set them by hand on instances. See Operator overloading.

__index: reading a missing key

Runs when you read a key the table does not actually have. It can be either a table or a function.

  • If __index is a table, the missing key is looked up in that table instead. This is how inheritance-style fallback works.
  • If __index is a function, it is called as __index(t, key) and whatever it returns becomes the result. Return nil for "no value".
local defaults = { color = "black" }
local a = setmetatable({}, { __index = defaults })
print(a.color)   -- black   (not in a, fell back to defaults)

local b = setmetatable({}, {
  __index = function(t, key) return `missing: {key}` end,
})
print(b.anything)   -- missing: anything

__newindex: assigning a missing key

Runs when you assign to a key the table does not already have. It is called as __newindex(t, key, value). The plain assignment does not happen on its own; your function decides what to do (store it elsewhere, validate it, reject it). To store it on the original table anyway, use rawset(t, key, value), which bypasses the metamethod.

local readonly = setmetatable({}, {
  __newindex = function(t, key, value)
    print(`blocked write to {key}`)
  end,
})
readonly.x = 1   -- blocked write to x  (x is never actually set)

__call: calling the table

Runs when you use the table as if it were a function: t(arg1, arg2). It is called as __call(t, arg1, arg2, ...), with the table itself as the first argument, and its return values become the call's results.

local adder = setmetatable({ base = 10 }, {
  __call = function(self, n) return self.base + n end,
})
print(adder(5))   -- 15

__tostring: turning the table into text

Runs when tostring(t) or print(t) needs a string for the table. It is called as __tostring(t) and must return a string.

local point = setmetatable({ x = 1, y = 2 }, {
  __tostring = function(t) return `({t.x}, {t.y})` end,
})
print(point)   -- (1, 2)

__type: naming the table's type

Read by the TypeOf(v) builtin (the plain type(v) builtin ignores metatables and still says "table"). It may be a plain string, or a function called as __type(t) that returns a string.

local vec = setmetatable({}, { __type = "Vector" })
print(TypeOf(vec))   -- Vector
print(type(vec))     -- table

__len: the length operator

Runs when you take the length with #t. It is called as __len(t) and should return a number.

local bag = setmetatable({ count = 7 }, {
  __len = function(t) return t.count end,
})
print(#bag)   -- 7

__eq, __lt, __le: comparison

These let two tables be compared.

  • __eq(a, b) runs for a == b (and ~= is its negation). It only applies when both sides are tables; comparing a table to anything else is plain false. Return a boolean.
  • __lt(a, b) runs for a < b.
  • __le(a, b) runs for a <= b.

Note that on plain tables only < and <= consult the metatable. The flipped operators > and >= do not fall back to __lt/__le — a > b between tables is a run-time error ("cannot compare table and table"), so write b < a instead. (Class instances are more convenient: for them > and >= are derived automatically by swapping the operands of operator < / operator <=. See Classes.)

local meta = {
  __eq = function(a, b) return a.id == b.id end,
  __lt = function(a, b) return a.id < b.id end,
}
local a = setmetatable({ id = 1 }, meta)
local b = setmetatable({ id = 2 }, meta)
print(a < b)   -- true
print(b < a)   -- false  (use this instead of a > b)

Arithmetic and concatenation

These run when a table is used with an arithmetic operator or ... Each is called as f(a, b) with the two operands; either operand may be the one carrying the metamethod. Return the result value.

MetamethodOperatorCalled as
__adda + b__add(a, b)
__suba - b__sub(a, b)
__mula * b__mul(a, b)
__diva / b__div(a, b)
__moda % b__mod(a, b)
__powa ^ b__pow(a, b)
__concata .. b__concat(a, b)
__unm-a (unary minus)__unm(a)
local Vec = {
  __add = function(a, b) return setmetatable({ x = a.x + b.x }, getmetatable(a)) end,
  __unm = function(a) return setmetatable({ x = -a.x }, getmetatable(a)) end,
}
local p = setmetatable({ x = 1 }, Vec)
local q = setmetatable({ x = 4 }, Vec)
print((p + q).x)   -- 5

Raw access: bypassing metamethods

Sometimes you want to touch the table directly and ignore its metatable. The raw functions do exactly that: rawget(t, key) reads without __index, rawset(t, key, value) writes without __newindex, rawlen(t) takes the length without __len, and rawequal(a, b) compares without __eq. There is no __metatable protection field: getmetatable always returns the real metatable.

The host program can attach native (Rust) functions as metamethods, so behavior like a custom __index can be driven from outside the language. See the Host API.
