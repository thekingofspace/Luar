Tables

Tables are the general-purpose structure: arrays, maps, and objects are all tables. Arrays are 1-based.

local array = { 10, 20, 30 }
local map = { name = "luar", version = 1 }
local mixed = { 1, 2, [10] = "ten", label = "x" }

print(array[1])     -- 10
print(map.name)     -- "luar"  (sugar for map["name"])
print(#array)       -- 3

Metatables

A metatable is a second table that controls how the first table behaves. By itself a plain table just stores keys and values. Attach a metatable to it, and you can change what happens when someone reads a missing key, calls the table, turns it into a string, or uses it with an operator like +.

Set a metatable with setmetatable(t, meta) and read it back with getmetatable(t). The special fields you put in the metatable are called metamethods, and their names all begin with two underscores.

local base = { greet = function() return "hi" end }
local obj = setmetatable({}, { __index = base })
print(obj.greet())   -- found through __index

Metamethods in full

Each metamethod below lists when it runs, the arguments your function receives, and what it should return. Classes are built on this machinery: when you write operator + or operator tostring in a class, those become the matching metamethods for you, so you rarely set them by hand on instances. See Operator overloading.

__index: reading a missing key

Runs when you read a key the table does not actually have. It can be either a table or a function.

  • If __index is a table, the missing key is looked up in that table instead. This is how inheritance-style fallback works.
  • If __index is a function, it is called as __index(t, key) and whatever it returns becomes the result. Return nil for "no value".
local defaults = { color = "black" }
local a = setmetatable({}, { __index = defaults })
print(a.color)   -- black   (not in a, fell back to defaults)

local b = setmetatable({}, {
  __index = function(t, key) return `missing: {key}` end,
})
print(b.anything)   -- missing: anything

__newindex: assigning a missing key

Runs when you assign to a key the table does not already have. It is called as __newindex(t, key, value). The plain assignment does not happen on its own; your function decides what to do (store it elsewhere, validate it, reject it). To store it on the original table anyway, use rawset(t, key, value), which bypasses the metamethod.

local readonly = setmetatable({}, {
  __newindex = function(t, key, value)
    print(`blocked write to {key}`)
  end,
})
readonly.x = 1   -- blocked write to x  (x is never actually set)

__call: calling the table

Runs when you use the table as if it were a function: t(arg1, arg2). It is called as __call(t, arg1, arg2, ...), with the table itself as the first argument, and its return values become the call's results.

local adder = setmetatable({ base = 10 }, {
  __call = function(self, n) return self.base + n end,
})
print(adder(5))   -- 15

__tostring: turning the table into text

Runs when tostring(t) or print(t) needs a string for the table. It is called as __tostring(t) and must return a string.

local point = setmetatable({ x = 1, y = 2 }, {
  __tostring = function(t) return `({t.x}, {t.y})` end,
})
print(point)   -- (1, 2)

__len: the length operator

Runs when you take the length with #t. It is called as __len(t) and should return a number.

local bag = setmetatable({ count = 7 }, {
  __len = function(t) return t.count end,
})
print(#bag)   -- 7

__eq, __lt, __le: comparison

These let two tables be compared.

  • __eq(a, b) runs for a == b (and the opposite for ~=). It only applies when both sides are tables. Return a boolean.
  • __lt(a, b) runs for a < b. The greater-than form a > b is handled by swapping the operands, so you do not write a separate one.
  • __le(a, b) runs for a <= b, and >= is derived the same way.
local meta = {
  __eq = function(a, b) return a.id == b.id end,
  __lt = function(a, b) return a.id < b.id end,
}
local a = setmetatable({ id = 1 }, meta)
local b = setmetatable({ id = 2 }, meta)
print(a < b, a > b)   -- true  false

Arithmetic and concatenation

These run when a table is used with an arithmetic operator or ... Each is called as f(a, b) with the two operands; either operand may be the one carrying the metamethod. Return the result value.

MetamethodOperatorCalled as
__adda + b__add(a, b)
__suba - b__sub(a, b)
__mula * b__mul(a, b)
__diva / b__div(a, b)
__moda % b__mod(a, b)
__powa ^ b__pow(a, b)
__concata .. b__concat(a, b)
__unm-a (unary minus)__unm(a)
local Vec = {
  __add = function(a, b) return setmetatable({ x = a.x + b.x }, getmetatable(a)) end,
  __unm = function(a) return setmetatable({ x = -a.x }, getmetatable(a)) end,
}
local p = setmetatable({ x = 1 }, Vec)
local q = setmetatable({ x = 4 }, Vec)
print((p + q).x)   -- 5

Raw access: bypassing metamethods

Sometimes you want to touch the table directly and ignore its metatable. The raw functions do exactly that: rawget(t, key) reads without __index, rawset(t, key, value) writes without __newindex, rawlen(t) takes the length without __len, and rawequal(a, b) compares without __eq.

The host program can attach native (Rust) functions as metamethods, so behavior like a custom __index can be driven from outside the language. See the Host API.
