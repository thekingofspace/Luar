Variables

A variable is a name that holds a value. You create one by declaring it, then read or change it by name. This page covers how to declare variables, the kinds of values they can hold, and how visibility works.

Values and types

Every value in LUAR has a type. A variable is not tied to a type; it just holds whatever value you put in it, and that value carries its own type. These are the value types:

TypeWhat it isExamples
nilThe absence of a value. The default for anything not set.nil
booleanA true or false value.true, false
numberAn integer or a floating-point number.42, -7, 3.14
stringText. See Strings for the three forms."hi", `x={n}`
tableThe general structure: arrays, maps, and objects. See Tables.{ 1, 2 }
functionA piece of code you can call. See Functions.function() end
class instanceAn object made from a class. See Classes.Point(1, 2)

You can ask a value its type at run time with type(value), which returns a string like "number" or "table".

print(type(1))         -- "number"
print(type("hi"))      -- "string"
print(type({}))        -- "table"
print(type(nil))       -- "nil"

An enum and a class report their own names rather than "table", so you can tell them apart from a plain table:

enum Color { Red Green Blue }
class Point { public x: number = 0 }
print(type(Color))   -- "enum"
print(type(Point))   -- "class"

Two values are special when used as a condition (in an if or a while): false and nil are "falsy", and everything else, including 0 and the empty string, is "truthy".

You can convert between strings and numbers with tostring(value) and tonumber(text).

local, const, and bare assignment

There are two kinds of binding: mutable (reassignable) and immutable. Only local creates a mutable one — every other way of declaring defaults to immutable, including const, a bare assignment to a new name, a pub declaration with no other keyword, and function declarations.

local count = 0      -- mutable: reassignment is fine
count = count + 1

const PI = 3.14159  -- immutable

greeting = "hi"        -- a bare new name declares an immutable local
pub version = 1        -- no `local` keyword, so also immutable

Reassigning an immutable binding fails at run time with cannot assign to const 'name' (only `nil` may free it), and the Ferrite checker flags it while you type (MutateImmutable). The one exception is nil: assigning nil to an immutable binding is allowed and frees the value it held.

const handle = openThing()
handle = nil   -- allowed: releases the value
handle = 5     -- run-time error: cannot assign to const 'handle'

Compound assignment (+=, -=, *=, /=, %=, ..=) reads, combines, and writes back in one statement, so it requires a mutable binding too.

local msg = "a"
msg ..= "b"     -- msg is now "ab"

Type annotations

Any declaration may carry a type after the name. Annotations are used by the editor for checking, hovers, and completion, and are erased before the script runs — they never change behaviour.

local age: number = 30
const NAME: string = "luar"

pub and export

pub promotes a binding to the shared global scope. In a module a pub binding is visible to every file that requires it; without it, the binding stays private to the file. Writing export before a declaration does the same thing as pub.

pub const VERSION = "1.0"   -- visible everywhere once required
const SECRET = "hidden"      -- private to this file
pub local counter = 0       -- shared and reassignable
export const NAME = "luar"  -- same as pub const

Multiple declarations

One statement can declare several names. A declaration always needs an initializer — local a on its own is a syntax error; write local a = nil if you really want an empty slot. Values are matched to names left to right: names without a value get nil, and extra values are discarded. A function call (or ...) in the last position supplies all of its values.

local a, b, c = 1, 2     -- c is nil
local lo, hi = minmax(9, 4)   -- both returns are used

Assignment to already-declared mutable variables works the same way, which gives the classic swap idiom — every value on the right is evaluated before anything is stored:

local x, y = 1, 2
x, y = y, x             -- x = 2, y = 1

Scope and shadowing

Each block introduces a new scope. An inner declaration that reuses an outer name shadows it (Ferrite warns with ShadowedVariable). See Scope for the full rules.
