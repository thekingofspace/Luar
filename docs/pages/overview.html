LUAR

LUAR is a small, class-based scripting language inspired by Lua and Luau. It has a tree-walking interpreter written in Rust, a static checker called Ferrite that runs in your editor, real asynchronous coroutines, a garbage collector, a precompilation format, and a host embedding API for Rust programs.

-- A documented function. Hover it in the editor to read this text.
-- @param name the person to greet
-- @return string
function greet(name)
  return "hello, " .. name
end

class Animal {
  public name: string = "animal"
  constructor(name) self.name = name end
  function describe() return self.name end
}

final class Dog extends Animal {
  override function describe()
    return super:describe() .. " woof"
  end
}

enum State { Idle Running Done }

const rex = Dog("Rex")
print(greet(rex.name))
print(rex:describe())
print(State.Running)

Design at a glance

  • Dynamically typed at run time. Type annotations are parsed for tooling and then scrubbed before execution, so they cost nothing at run time.
  • Class-based objects with single inheritance, mixins, interfaces, operator overloading, getters and setters, and access control.
  • Modules are files. pub declarations are shared across modules; everything else stays private to the file that declared it.
  • Runtime errors are caught with pcall, which returns false and the error message instead of crashing.
  • The whole language runs on the interpreter, including precompiled code, so every feature is available everywhere.

Highlights

  • Classes. Single inheritance with extends, mixin composition, implements contracts, public/private/protected/static members, abstract/final/override modifiers, operator overloading, getters and setters, and destructors that run deterministically when the last reference to an instance goes away.
  • Static checking in the editor. The Ferrite checker runs over twenty lint checks on every edit, and the language server adds type inference, inlay hints, and its own diagnostics — all without a compile step, since checking happens on the source while the runtime stays dynamic. See Type rules.
  • Real async coroutines. When a host-provided native blocks (a wait, a file read), the coroutine detaches and keeps running on its own thread while the main script continues; the host drains stragglers before exit. Scripted coroutines use the familiar coroutine.create/resume/yield API from the standard library.
  • Proper tail calls. A return f(x) whose value is a single call reuses the current stack frame, so loops written as recursion — including mutual recursion — never overflow the stack. See Functions.
  • Memory you can steer. A garbage collector handles the common case, while buff declarations give fixed-size, manually freed slots when you want control.
  • Precompilation. Pack a program to a binary .luarb file to ship without source text and skip parsing at load time; it runs on the same interpreter with every feature intact.
  • Embeddable in Rust. A host can register native functions, define native classes, call script functions, launch them as background coroutines, and trace any value back to the script file that created it.
  • Modules. require with relative paths, @alias names from a luari.json, precompiled module fallback, caching, and circular-require detection.
