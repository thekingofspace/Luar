LUAR

LUAR is a small, class-based scripting language inspired by Lua and Luau. It has a tree-walking interpreter written in Rust, a static checker called Ferrite, a garbage collector, a precompilation format, and a host embedding API.

-- A documented function. Hover it in the editor to read this text.
-- @param name the person to greet
-- @return string
function greet(name)
  return "hello, " .. name
end

class Animal {
  public name: string = "animal"
  constructor(name) self.name = name end
  function describe() return self.name end
}

final class Dog extends Animal {
  override function describe()
    return super:describe() .. " woof"
  end
}

enum State { Idle Running Done }

const rex = Dog("Rex")
print(greet(rex.name))
print(rex:describe())
print(State.Running)

Design at a glance

  • Dynamically typed at run time. Type annotations are parsed for tooling and then scrubbed before execution, so they cost nothing at run time.
  • Class-based objects with single inheritance, mixins, interfaces, operator overloading, getters and setters, and access control.
  • Modules are files. pub declarations are shared across modules; everything else stays private to the file that declared it.
  • Errors are values you catch with pcall.
  • The whole language runs on the interpreter, including precompiled code, so every feature is available everywhere.
