Functions

Declaring functions

A plain function declaration creates an immutable binding (the same as const). Prefix it with local to get a binding you can reassign later, or with pub to share it with other files (see Variables).

function add(a, b)      -- immutable binding, like const
  return a + b
end

local function helper()  -- reassignable binding
  return 1
end

pub function shared()    -- visible to files that require this one
end

Functions are ordinary values, so the anonymous form can be stored anywhere a value goes:

local double = function(n) return n * 2 end

Defining functions inside tables

A dotted name after function assigns the function into an existing table. A colon instead of the final dot adds an implicit first parameter named self, so the function can be called with method syntax (t:name(...)).

local shapes = { pi = 3.14159 }

function shapes.area(r)        -- same as shapes.area = function(r) ... end
  return shapes.pi * r * r
end

function shapes:describe()      -- implicit `self` parameter
  return `a circle of radius 2 has area {self.area(2)}`
end

print(shapes.area(2))
print(shapes:describe())

The dotted path can be deeper than one level (function a.b.c()), but every table on the path must already exist — this form is an assignment, not a declaration.

Parameters and arguments

Calls adjust their arguments to the parameter list: missing arguments become nil, and extra arguments are dropped (unless the function is variadic, below). Parameters are fresh mutable locals on every call. Type annotations on parameters and return values are for the editor only and are erased before execution.

function greet(name: string, punct: string): string
  if punct == nil then punct = "!" end   -- default-argument idiom
  return `hi {name}{punct}`
end

print(greet("sam"))   -- punct arrives as nil

Closures

local make = function(start)
  local n = start
  return function()
    n += 1
    return n
  end
end

local next = make(10)
print(next())   -- 11
print(next())   -- 12

The inner function keeps the variables of the scope it was created in alive — see Scope for how capture works.

Multiple returns

A return may carry several values, separated by commas.

function minmax(a, b)
  if a < b then return a, b end
  return b, a
end

local lo, hi = minmax(9, 4)   -- lo = 4, hi = 9

How many of those values survive depends on where the call appears:

  • In the last position of an expression list — a declaration, an assignment, the arguments of another call, a return, or a table constructor — the call expands to all of its values.
  • Anywhere else (the middle of a list, an arithmetic expression), it is cut down to its first value.
  • Wrapping a call in parentheses, (f()), also truncates it to one value.
  • Missing values become nil; extra values are discarded.
local a, b = minmax(1, 2), 99   -- a = 1 (truncated), b = 99
local t = { minmax(1, 2) }       -- { 1, 2 } (expands in last position)

Varargs

End a parameter list with ... to accept any number of extra arguments. Inside the body, ... stands for those extras: in the last position of a list it expands to all of them, elsewhere it yields the first one. { ... } collects them into a table, and select("#", ...) counts them. A type for the extras may be written as ...: Type (erased at run time).

function sum(...)
  local total = 0
  for _, n in ipairs({ ... }) do total += n end
  return total
end

function count(...)
  return select("#", ...)   -- number of extra arguments
end

Proper tail calls

A return f(args) whose value is exactly one call to a Luar function is executed as a tail call: the current frame is discarded before f runs, so recursion in tail position uses constant stack no matter how deep it goes. All of the called function's return values are passed through unchanged.

local function count(n, total)
  if n == 0 then return total end
  return count(n - 1, total + 1)
end

count(10000000, 0)  -- runs in constant stack space

Only plain calls qualify: return f(x) + 1 and return a, f(x) are ordinary returns, and a tail position call to something that is not a Luar function (a native function or a class constructor) is simply evaluated normally. Because the calling frame is gone before the tail call runs, its scoped destructors fire before f executes rather than after it returns.

Call sugar

When a function takes a single string, interpolated string, or table literal, the parentheses are optional.

print "hello"              -- print("hello")
print `got {n}`            -- print(`got {n}`)
configure { debug = true }   -- configure({ debug = true })

Generic parameters

local function identity<T>(x: T): T
  return x
end

Generic parameters, like all type annotations, are used by the editor and scrubbed before execution.

Documentation comments

Comment lines written directly above a declaration become its documentation: the editor shows them in the hover box when you point at the name (and at members of classes). Consecutive -- lines are collected until a non-comment line or a --# directive line is reached, and the text is rendered as Markdown, so emphasis like **bold** works.

-- Clamp a number into the range **[lo, hi]**.
-- Values below lo become lo; values above hi become hi.
function clamp(value, lo, hi)
  if value < lo then return lo end
  if value > hi then return hi end
  return value
end
