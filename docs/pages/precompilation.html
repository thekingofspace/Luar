Precompilation

Precompilation is optional. LUAR runs .luar source directly — you never have to precompile anything to use any feature. It exists for when you want to load or ship a program without its source text, or skip parsing at startup. If you only ever run source, you can ignore this page.

What a .luarc is

A .luarc file is a precompiled program. The source has been tokenized, parsed, and had its type annotations scrubbed, and the resulting syntax tree is packed into a compact binary. It is not machine code and not a separate runtime: it is the very same program the parser would hand to the interpreter, frozen to bytes. Loading one skips the lexer and parser, and from there everything runs on the ordinary interpreter — classes, enums, mixins, operators, and the standard library all behave exactly as they do from source.

From the runner

The runner can precompile a file and run the result.

compile game.luar game.luarc   # produce a .luarc
run game.luarc                 # run it, just like running the source

From Rust

precompile_source turns source into the packed bytes, and run_precompiled loads those bytes and runs them in a fresh interpreter. You decide where the bytes live — a file on disk, an asset embedded in your binary, or a payload over the network.

let bytes = luar::precompile_source(src)?;     // Vec<u8>, the packed program
std::fs::write("game.luarc", &bytes)?;          // optional: save it somewhere

let bytes  = std::fs::read("game.luarc")?;
let interp = luar::run_precompiled(&bytes)?;    // load and run; returns the interpreter

These are thin wrappers over precompile::pack (syntax tree to bytes) and precompile::unpack (bytes back to a syntax tree), which you can call directly if you already hold a parsed program.

When to reach for it

  • Ship without source. Hand out .luarc bytes when you would rather not distribute the original .luar text.
  • Faster startup. Large programs skip lexing and parsing on load, which adds up when a host boots many scripts at once.
  • Embedding. Bake the bytes straight into your host binary and run them with run_precompiled, no files required.

Because annotations are scrubbed when the program is packed, a .luarc carries no type information; checking happens earlier, on the source, with Ferrite. The binary is portable as long as the LUAR that reads it understands the format it was written with.
