Control flow

if / elseif / else

if score > 90 then
  print("A")
elseif score > 80 then
  print("B")
else
  print("C")
end

Every value is truthy except false and nil.

while and for

while n > 0 do
  n -= 1
end

for i = 1, 10 do print(i) end
for i = 10, 1, -1 do print(i) end   -- explicit step

for index, value in ipairs(list) do print(index, value) end
for key, value in pairs(map) do print(key, value) end

Name an unused loop variable _ to silence UnusedLoopVariable. break exits the innermost loop.

Small numeric for loops with constant integer bounds (up to 16 iterations) are unrolled automatically when the script loads, removing the per-iteration loop overhead. Loops containing break or closures that capture the loop variable keep their normal form, so behaviour never changes — only speed.

switch

switch is an expression. The first case whose value equals the subject runs, and a return inside it becomes the value of the switch. An optional default runs when no case matches. Each case has its own end, and the whole switch is closed by a final end.

local label = switch(state)
  case "idle"
    return "waiting"
  end
  case "run"
    return "working"
  end
  default
    return "unknown"
  end
end

Why switch and case instead of a chain of elseif

You could write the same logic as a long if / elseif chain, but a switch is usually the better tool when you are branching on the value of one thing.

  • It states your intent. A switch says "I am choosing a branch based on this one value." A pile of elseif lines does not; a reader has to check that every condition really tests the same subject.
  • The subject is written once. With elseif you repeat state == on every line, which is easy to get wrong (a typo can silently test the wrong variable). A switch names the subject a single time.
  • It is an expression. A switch can be assigned straight to a variable or returned, so you do not need a temporary variable that each branch reassigns.
  • The editor helps more. Because the subject's type is known, the editor can suggest the values to match (for example the variants of an enum or the allowed strings of a literal type), and the checker can flag a duplicate case.

Reach for if / elseif when the branches test genuinely different conditions (different variables, ranges, or compound expressions). Reach for switch when you are comparing one value against a set of possibilities.
