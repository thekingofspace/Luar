Control flow

if / elseif / else

if score > 90 then
  print("A")
elseif score > 80 then
  print("B")
else
  print("C")
end

Conditions are checked top to bottom and the first truthy one runs. Every value is truthy except false and nil — including 0 and the empty string.

while

The condition is tested before each pass, so a body may run zero times.

while n > 0 do
  n -= 1
end

Numeric for

for var = start, stop, step counts from start to stop inclusive. The step defaults to 1 and may be negative; all three must be numbers, and a step of 0 is a run-time error. The loop variable is scoped to the loop and does not exist after it.

for i = 1, 10 do print(i) end
for i = 10, 1, -1 do print(i) end   -- explicit step

Small numeric for loops with constant integer bounds and step (up to 16 iterations) are unrolled automatically when the script loads, removing the per-iteration loop overhead. Loops containing break or anything that creates a function keep their normal form, so behaviour never changes — only speed.

Generic for

for a, b in ... drives an iterator. The expression list after in supplies an iterator function, a state value, and an initial control value; the iterator is called each pass and the loop stops when its first result is nil. In practice you get these triples from ipairs (array part, in order) and pairs (every key).

for index, value in ipairs(list) do print(index, value) end
for key, value in pairs(map) do print(key, value) end

Name an unused loop variable _ to silence UnusedLoopVariable.

break

break exits the innermost enclosing loop immediately. There is no continue statement; skip an iteration with an if around the rest of the body. Ferrite reports a break outside any loop as an error (BreakOutsideLoop).

switch

switch is an expression. The subject is written once, in parentheses; the first case whose value equals the subject runs, and a return inside it becomes the value of the switch. An optional default runs when no case matches. Each case has its own end, and the whole switch is closed by a final end.

local label = switch(state)
  case "idle"
    return "waiting"
  end
  case "run"
    return "working"
  end
  default
    return "unknown"
  end
end

The rules, in detail:

  • First match wins, no fallthrough. Cases are independent blocks; once one runs, the switch is finished.
  • return yields the switch's value — it does not return from the enclosing function. Only the first returned value is kept. A case that runs without reaching a return produces nil, as does a subject that matches nothing when there is no default.
  • Patterns are expressions. A case pattern can be any expression (case 2 + 3, case Color.Red); it is compared to the subject with the same semantics as == (numbers compare across int/float, strings by content, tables by identity).
  • Each case body is its own scope, so locals declared in one case are invisible to the others.
  • It works as a statement too — you may use a switch purely for its side effects and ignore the value.
switch(command)        -- statement position: value is discarded
  case "save"
    save()
  end
  case "quit"
    shutdown()
  end
end

Performance: when every case pattern is a literal (an integer, string, boolean, or whole float), the interpreter builds a jump table the first time the switch runs and reuses it, so matching is a single hash lookup instead of a walk down the cases. Whole-float subjects normalize, so pick(2.0) still hits case 2. If any pattern is not a literal, the cases are simply evaluated top to bottom until one matches.

Why switch and case instead of a chain of elseif

You could write the same logic as a long if / elseif chain, but a switch is usually the better tool when you are branching on the value of one thing.

  • It states your intent. A switch says "I am choosing a branch based on this one value." A pile of elseif lines does not; a reader has to check that every condition really tests the same subject.
  • The subject is written once. With elseif you repeat state == on every line, which is easy to get wrong (a typo can silently test the wrong variable). A switch names the subject a single time.
  • It is an expression. A switch can be assigned straight to a variable or returned, so you do not need a temporary variable that each branch reassigns.
  • The editor helps more. Because the subject's type is known, the editor can suggest the values to match (for example the variants of an enum or the allowed strings of a literal type), and the checker can flag a duplicate case.

Reach for if / elseif when the branches test genuinely different conditions (different variables, ranges, or compound expressions). Reach for switch when you are comparing one value against a set of possibilities.
